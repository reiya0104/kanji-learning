# ADR 0001: 技術スタック

## ステータス
Accepted

## コンテキスト
iPad / iPhone での利用を前提とした個人用学習アプリを作る。
毎日継続して使うため、インストール・起動が簡単なネイティブアプリが望ましい。
開発者はTypeScriptに慣れており、単独で開発・保守する。

## 決定
React Native + TypeScript を採用する。

## 理由
- iOS / Android の両プラットフォームを単一コードベースでカバーできる
- TypeScript により domain 層の型安全性を確保できる
- Jest によるユニットテストが容易に構築できる
- 将来的に Expo への移行も検討できる

## トレードオフ
- Flutter の採用は見送る（Dart の学習コストが高い）
- Web アプリ（PWA）は見送る（iOS でのオフライン体験が劣る）
- Go / Rust はロジック分離の将来候補として検討するが、初期は対象外とする
