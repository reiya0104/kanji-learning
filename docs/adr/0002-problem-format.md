# ADR 0002: 問題データ形式

## ステータス
Accepted

## コンテキスト
学習問題データをどこにどのような形式で保持するか決める必要がある。
初期段階では問題数が少なく、外部インフラを持たない個人プロジェクトである。

## 決定
問題データを JSON ファイルで管理する。スキーマは `src/domain/problem.ts` の Problem 型に従う。

## 理由
- TypeScript の型と JSON スキーマを一致させることで、型チェックが問題データの整合性を保証する
- ファイルベースなので Git での変更管理が容易
- 外部 DB 不要で、オフライン動作と相性が良い

## トレードオフ
- 問題数が増えた場合（目安：1,000件以上）はデータベースへの移行を検討する
- 初期は外部 DB を使わずローカルファイルで完結させる（同期機能は v0.3 以降）
