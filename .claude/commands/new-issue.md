以下の手順で GitHub Issue を作成してください。

## ステップ1：情報収集

まだ情報が与えられていない場合は、以下を確認・質問してください：
- **タイトル**：Issue のタイトル
- **フェーズ**：phase:0 / phase:1 / phase:2 / phase:3 / harness（ラベル）
- **マイルストーン**：どのフェーズに属するか（harness は不要）

## ステップ2：内容をドラフト

以下のテンプレートで内容をドラフトし、ユーザーに確認してください。

```
## 背景
（なぜこの Issue が必要か。How は書かない）

## やること
- [ ] （What のみ。実装方法は書かない）

## 完了条件（DoD）
- [ ] （検証可能な完了基準）

## スコープ外
- （今回やらないこと）
```

## ステップ3：Issue 作成

ユーザーの確認後、以下の形式で作成してください：

```bash
gh issue create \
  --title "タイトル" \
  --label "ラベル" \
  --milestone "マイルストーン名" \
  --body $'## 背景\n\n内容\n\n## やること\n\n- [ ] 項目\n\n## 完了条件（DoD）\n\n- [ ] 項目\n\n## スコープ外\n\n- 項目'
```

**注意**：body は必ず `$'...'` 構文を使うこと（`\n` を実際の改行として扱うため）。

## ステップ4：Project に追加

```bash
gh project item-add 6 --owner reiya0104 --url "作成された Issue の URL"
```
