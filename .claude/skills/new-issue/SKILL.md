以下の手順で GitHub Issue を作成してください。

## ステップ1：情報収集

まだ情報が与えられていない場合は、以下を確認・質問してください：
- **タイトル**：Issue のタイトル
- **フェーズ**：phase:0 / phase:1 / phase:2 / phase:3 / harness（ラベル）
- **マイルストーン**：どのフェーズに属するか（harness は不要）
- **依存関係**：他の Issue が完了していないと着手できない場合は Issue 番号を確認する

## ステップ2：内容をドラフト

以下のテンプレートで内容をドラフトし、ユーザーに確認してください。
依存関係がない場合は「依存関係」セクションを省略してよい。

**複雑チケットの判定**：以下のいずれかに該当する場合は「複雑」と判断し、背景セクションの先頭に警告文を挿入すること。
- 実装方法（How）が自明でない、または複数の設計案が考えられる
- 複数のコンポーネント・レイヤーにまたがる変更が必要
- 未知の技術・ツールの調査・導入を含む

該当する場合、背景セクションの先頭に以下を挿入する：
```
⚠️ 作業前に **Plan モード**（`/plan`）で手順を設計・確認してから進めること。
```

```
## 背景
（なぜこの Issue が必要か。How は書かない）

## やること
- [ ] （What のみ。実装方法は書かない）

## 完了条件（DoD）
- [ ] （検証可能な完了基準）

## 依存関係
- Depends on #XX （この Issue が完了していないと着手できない）

## スコープ外
- （今回やらないこと）
```

**依存関係の記法ルール**
- `Depends on #XX`：この Issue を始める前に #XX が完了している必要がある
- GitHub のクロスリファレンス機能により #XX のページにも自動で言及が表示される
- 依存がない場合はセクションごと省略する

## ステップ3：Issue 作成

ユーザーの確認後、以下の形式で作成してください：

```bash
gh issue create \
  --title "タイトル" \
  --label "ラベル" \
  --milestone "マイルストーン名" \
  --assignee "@me" \
  --body $'## 背景\n\n内容\n\n## やること\n\n- [ ] 項目\n\n## 完了条件（DoD）\n\n- [ ] 項目\n\n## 依存関係\n\n- Depends on #XX\n\n## スコープ外\n\n- 項目'
```

**注意**：body は必ず `$'...'` 構文を使うこと（`\n` を実際の改行として扱うため）。依存関係がない場合は該当セクションごと省略する。

## ステップ4：Project に追加

```bash
gh project item-add 6 --owner reiya0104 --url "作成された Issue の URL"
```
